# تحديث المناصب (3 مستويات) + تقييد السوبر حسب المديرية

هذه الحزمة مبنية على النسخة المستقرة التي تعمل مع **Support Mode**، وتم تطويرها لإضافة 3 مستويات للصلاحيات مع الحفاظ على الدعم.

## 1) المناصب الجديدة
- **super_admin (السوبر أدمن)**: كل الصلاحيات + دعم المدارس + إدارة (السوبر) و (الأدمن).
- **super (السوبر - مديرية)**: لا يملك Support Mode، ويستطيع فقط إضافة/تعديل **الأدمن** داخل نطاق مديريته.
- **admin (الأدمن - مدرسة)**: إدارة مدرسة واحدة فقط.

> ملاحظة: تمت المحافظة على التوافق مع البيانات القديمة: إذا كان دور المستخدم في allowlist = `super` **بدون** حقل المديرية `governorate` فسيتم اعتباره `super_admin` (لعدم كسر النسخ القديمة).

## 2) السوبر أدمن الرئيسي
- تم تثبيت السوبر أدمن الرئيسي على البريد:
  - `3asal2030@gmail.com`
- لا يمكن للسوبر (المديرية) تعديل هذا الحساب إطلاقاً.
- الحد الأقصى للسوبر أدمن في الواجهة: **2 فقط** (الرئيسي + واحد يضيفه الرئيسي).  
  *(الحد مُطبق في الواجهة كحماية إضافية.)*

## 3) تقييد السوبر حسب المديرية (11 مديرية + "سوبر الوزارة")
- تمت إضافة قائمة المديريات المطلوبة داخل:
  - `src/constants/directorates.ts`
- عند إنشاء مستخدم بدور **super** ستظهر قائمة اختيار المديرية + خيار **سوبر الوزارة**.
- السوبر (المديرية) يرى فقط المدارس (Tenants) التابعة لمديريته داخل:
  - صفحة `system`
  - نافذة اختيار مدرسة عند "الدخول للبرنامج" (SuperProgramEnter)

## 4) الدعم (Support Mode)
- الدعم أصبح **للسوبر أدمن فقط**.
- تم إخفاء زر الدعم عن السوبر (المديرية) داخل صفحة `system`.

## 5) التغييرات التقنية (أهم الملفات)
- `src/constants/directorates.ts`  
  قائمة المديريات + دوال التوحيد normalizeText.
- `src/auth/AuthContext.tsx`  
  إضافة أدوار `super_admin / super / admin` + حقول governorate + booleans (isSuper, isAdmin, canSupport).
- `src/auth/ProtectedRoute.tsx`  
  إضافة `SystemRoute` للسماح بدخول `system` لكل من السوبر أدمن والسوبر.
- `src/pages/AdminSystem.tsx`  
  - تصفية المدارس حسب المديرية
  - إضافة اختيار المديرية عند إنشاء سوبر
  - منع السوبر من إنشاء أدوار أعلى
  - إخفاء الدعم عن السوبر
- `src/pages/SuperProgramEnter.tsx`  
  نافذة اختيار المدرسة أصبحت تُصفّي المدارس للسوبر حسب المديرية.
- `functions/src/index.ts`  
  تحديث roles + إضافة claim `governorate` + منع السوبر من Support Mode + تقييد إضافته للأدمن داخل نطاق المديرية.
- `firestore.rules`  
  تحديث helper functions لدعم roles الجديدة + السماح بقراءة المدارس للسوبر حسب governorate.
