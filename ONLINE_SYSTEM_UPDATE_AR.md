# تحديث النظام الكامل متعدد المستخدمين

تمت إضافة مرحلة جديدة لتحويل النظام إلى نسخة **Online Multi-User** بدل النسخة المحلية فقط.

## ما أضيف في هذه المرحلة

### 1) صفحة فريق العمل والصلاحيات
المسار:
- `/t/:tenantId/team-members`

الوظائف:
- إضافة عضو جديد للجهة
- تحديد الدور: مدير الجهة / مدير / تشغيل / مشاهد
- تفعيل أو إيقاف العضو
- حذف العضو
- مزامنة العضو محليًا + سحابيًا (Firestore) عند الإمكان
- تحديث وثيقة `allowlist` تلقائيًا عند إضافة العضو

### 2) صفحة إصدارات واعتماد التوزيع
المسار:
- `/t/:tenantId/task-distribution/versions`

الوظائف:
- حفظ إصدار جديد من التوزيع الحالي
- مزامنة التشغيل الحالي سحابيًا
- اعتماد التوزيع الحالي كنسخة رسمية
- استعادة أي إصدار سابق إلى الجدول الشامل
- حفظ بيانات الاعتماد محليًا + سحابيًا

### 3) تحسين الشريط الجانبي
- إضافة رابط **فريق العمل والصلاحيات**
- إضافة رابط **إصدارات واعتماد التوزيع**
- جعل التنقل Tenant-aware بحيث يفتح الروابط داخل `/t/:tenantId/...`

### 4) إضافة Route جديد مفقود
- `run-details`

## المجموعات الجديدة في Firestore
عند العمل أونلاين سيستخدم النظام هذه المسارات:

- `tenants/{tenantId}/members/{email}`
- `tenants/{tenantId}/distributionVersions/{versionId}`
- `tenants/{tenantId}/distribution/currentRun`
- `tenants/{tenantId}/distribution/approval`
- `allowlist/{email}`

## ملاحظات
- النظام يحافظ على fallback محلي داخل `localStorage` حتى لا يتعطل عند ضعف الاتصال أو عند عدم جاهزية القواعد السحابية.
- هذه المرحلة تضيف بنية قوية للنظام متعدد المستخدمين بدون كسر النسخة الحالية التي تعمل.
